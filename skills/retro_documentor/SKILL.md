# Universal Retro Documentor Engine: Absolute 1-23 Standard

The **Universal Retro Documentor Engine** is a stack-agnostic, hyper-intelligence system designed to autonomically reverse-engineer, synthesize, and document any codebase into a high-fidelity "System Archive."

---

## 🛠️ The Absolute 1-23 Universal Protocol

Every documentation artifact generated by this engine—whether it represents a **Project Root**, a **Module Directory**, or an individual **Source File**—MUST address all 23 sections of the systemic standard. **No subsets, shortcuts, or omissions are permitted.**

---

## 🦾 Autonomous Discovery Protocols (ADP)

Every documentation lifecycle follows these stack-agnostic discovery steps to find "Ground-Truth" for all 23 sections:

### 1. Entrance & Orchestration (Sections 1-7)
- **Web/Microservices**: Identify Controllers, Routes, and Logic Gateways.
- **Batch/Legacy**: Identify Jobs, Entry Points, and Schedulers.
- **Inversion**: Map technical code paths to high-level business functions.

### 2. Data & Infrastructure (Sections 8-14)
- **Persistence DNA**: Scan DDL (SQL), NoSQL schemas, and file-based structures.
- **Operational Scans**: Trace configuration, error handling, security models, and observability signals across the codebase.

### 3. Verification & Modernization (Sections 15-23)
- **Modernization Audit**: Standardized rubric for evaluating technical debt, coupling, and migration feasibility.
- **Reverse Engineering**: Infer logic assumptions and complexity wherever documentation is absent in the source.

---

## 📂 The Universal Archive Structure

- **Shadow Mirror Root**: All documentation resides in a `documentation/` shadow-root.
- **Shadow Tree Policy**: Root ➜ 1-23; Folder README ➜ 1-23; Source File .md ➜ 1-23.
- **Zero Placeholder Policy**: No "Minimal" or "N/A" summaries. If a feature is not explicit, the Documentor must provide a "Reverse Engineered Inference" based on the project's global context (Build files, configs, properties).

## 📊 Visual Engineering Standards (Mermaid)

- **Topology Diagram**: High-level component relationship.
- **Functional Call Trace**: Method-level sequence of primary business workflows.
- **Safety Syntax**: Quote all labels (`-- "Label" -->`) and place class definitions at the end.
